## Module 1: First Principles Breakdown

### Что именно мы решаем?

**Unknown (цель):** Создать прибыльный SaaS продукт, который позволяет российским B2B командам делать холодный email outreach с высокой доставляемостью в Яндекс/Mail.ru.

**Data (факты):**
- 60%+ российских B2B компаний используют Яндекс/Mail.ru как корпоративную почту
- Instantly.ai (глобальный лидер, $20M ARR) не прогревает Яндекс/Mail.ru и не принимает RU-платежи
- Ни один российский конкурент не имеет: warmup Яндекс + AI Reply Agent + multi-client + flat price одновременно
- Warmup работает через SMTP/IMAP peer network — технически это стандартные протоколы, реализуемые с нуля
- SAM РФ: $10–20M, CAGR 9.5%/год

**Условие:** Мы имеем технические знания, зрелый стек, понимание рынка, но нет продукта и клиентов.

### First Principles Разбор

Stripped to atoms:

```
Email Deliverability = f(Domain Reputation, IP Reputation, Content Quality, Technical Setup)

Domain Reputation = f(DMARC, SPF, DKIM) + f(Send Volume History) + f(Engagement Signals)

Engagement Signals = Открытия + Ответы + Перемещение из спама
                   = Реальное взаимодействие между аккаунтами (warmup pool)

Warmup Pool = N аккаунтов, которые:
  1. Отправляют письма ДРУГ ДРУГУ (SMTP)
  2. Открывают письма (IMAP read)  
  3. Перемещают из спама во входящие (IMAP move)
  4. Иногда отвечают (SMTP reply)
```

**Атомарная истина:** Warmup — это просто организованный обмен письмами между реальными почтовыми аккаунтами. Яндекс/Mail.ru поддерживают стандартные SMTP/IMAP. Технического барьера НЕТ — только организационный.

**Фундаментальная модель продукта:**
```
Пользователь добавляет аккаунт → Аккаунт входит в пул → 
Пул автоматически обменивается письмами →
Репутация растёт → Пользователь видит Inbox Score →
Пользователь запускает реальные кампании → Письма попадают во входящие
```

**Assumptions challenged:**
- ❌ "Нужна ML-модель для репутации" → Нет, достаточно эвристик: inbox_rate за 7/14/30 дней
- ❌ "Нужен большой warmup пул с самого старта" → Нет, даже 100 аккаунтов в пуле дают эффект
- ❌ "Нужно решать deliverability для всех ESP сразу" → Нет, фокус на Яндекс → потом Mail.ru → потом Gmail
- ❌ "AI Reply Agent сложен" → Claude API + простой state machine = 80% value за 20% усилий

## Module 2: Root Cause Analysis — 5 Whys

### Симптом: Российские B2B компании не могут делать эффективный холодный email outreach

**Why 1: Почему холодные письма попадают в спам?**
→ Потому что новые домены и почтовые аккаунты имеют низкую репутацию у ESP (Яндекс/Mail.ru)

**Why 2: Почему репутация низкая?**
→ Потому что нет истории взаимодействий: открытий, ответов, перемещений из спама

**Why 3: Почему нет истории взаимодействий?**
→ Потому что пользователи начинают сразу слать холодные письма, не прогрев аккаунты

**Why 4: Почему пользователи не прогревают аккаунты?**
→ Потому что инструменты прогрева либо не существуют для Яндекс/Mail.ru, либо дороги и сложны

**Why 5: Почему инструменты прогрева Яндекс/Mail.ru не существуют?**
→ **ROOT CAUSE:** Глобальные игроки (Instantly, Lemlist) не инвестируют в RU-специфичную инфраструктуру из-за санкций и малого относительного размера рынка. Российские игроки (Coldy) имеют базовый warmup, но не развивают AI и multi-client функционал из-за нехватки технических ресурсов.

### Root Cause Summary

```
ROOT CAUSE = Market Gap + Technical Accessibility Gap

Market Gap: Глобальные лидеры ушли из РФ → локальная ниша недоинвестирована
Technical Gap: Знание как строить warmup pool для Яндекс есть, но никто не собрал это
              вместе с AI Reply + multi-client + удобным UX + RU-платежами
```

**Если решить root cause:** Одна компания создаёт комплексный инструмент → захватывает рынок до прихода новых конкурентов → строит data moat (warmup pool с тысячами аккаунтов)

## Module 3: SCQA Framework

**Situation (стабильный контекст):**
Российский B2B рынок использует холодные письма как ключевой канал лидогенерации. Yandex.ru и mail.ru — доминирующие корпоративные почтовые провайдеры в РФ. До 2022 года компании использовали Instantly/Lemlist. После введения ограничений на платежи и Яндекс-специфичного спам-фильтра — эффективных инструментов нет.

**Complication (что изменилось/проблема):**
Instantly.ai недоступен для оплаты из РФ и не прогревает Яндекс/Mail.ru. Российские альтернативы (Coldy) имеют базовый warmup, но без AI Reply Agent, без multi-client режима, без flat-price для агентств. Рынок в $10–20M SAM растёт на 9.5%/год при практически нулевой конкуренции в full-stack.

**Question (ключевой вопрос):**
Как создать продукт, который станет очевидным выбором для SDR/Основателей и аутрич-агентств в РФ, монетизируется через flat SaaS pricing и строит data moat через warmup network?

**Answer (наша стратегия):**
Deliverability-first SaaS с 3 дифференциаторами:
1. **Warmup Яндекс/Mail.ru** (технический barrier to entry — нужен реальный пул аккаунтов)
2. **AI Reply Agent** (Claude API — снимает 2–4 ч/день рутины)  
3. **Multi-client агентский режим** (flat pricing — нет нигде в РФ)

Стратегия входа через Community-Led growth (Telegram + vc.ru) + freemium 7 дней → Aha Moment (первое письмо в inbox Яндекс за 10 мин) → конверсия.

## Module 4: Game Theory — Стратегические взаимодействия

### Игроки и мотивации

| Игрок | Цель | Ограничения | Информационные преимущества |
|-------|------|------------|--------------------------|
| **Поток** (мы) | Захватить $3–5M ARR в 3 года | Нет клиентов, нет warmup пула | Знание gap'ов конкурентов |
| **Coldy.ai** | Удержать #1 в РФ, монетизировать | Нет AI, нет eng ресурсов | Существующая база клиентов |
| **Respondo** | Выжить, найти нишу | Старый стек, мало фич | Данные о RU deliverability |
| **Instantly** | Вернуться в РФ когда возможно | Санкционные ограничения | Глобальная warmup сеть |
| **SDR/Основатель** | Получить pipeline без SDR команды | Ограниченный бюджет | Знает свою боль |
| **Агентство** | Масштабировать клиентов без найма | Нет multi-client инструмента | Понимает agency economics |

### Payoff Matrix: Стратегия входа

```
                    Coldy НЕ реагирует    Coldy добавляет AI
Мы входим дёшево:   [+15, -2]             [+8, +2]
Мы входим дорого:   [+5, -1]              [+3, +4]
```

**Nash Equilibrium:** Мы входим с ценой ниже Coldy (₽1,990 vs ₽2,500+), Coldy не реагирует немедленно (ROI реагирования низкий — нужно нанять AI-инженера). Оба выживают, мы берём весь новый спрос.

### Доминирующие стратегии

**Для Поток:**
1. **Ценообразование:** ₽1,990/мес (ниже Coldy) → снижает барьер первой конверсии
2. **Warmup пул:** Строить с первых 100 пользователей → data moat — Coldy не может скопировать быстро
3. **Community-First:** Telegram-канал + vc.ru контент ПЕРЕД запуском → доверие до показа продукта
4. **AI Reply:** Claude API интеграция — Coldy не имеет этого, на добавление им нужно 6–12 мес

**Сигнальная игра:**
- Публичный roadmap → показывает Coldy что мы серьёзны → снижает вероятность агрессивного ответа
- Партнёрство с amoCRM/Bitrix24 → integration lock-in перед тем как Coldy это сделает
- Open warmup pool invite → пользователи Coldy могут добавить свои аккаунты в наш пул → миграция данных

### Барьеры входа для будущих конкурентов

После 18 мес работы:
- **Data moat:** 5,000+ аккаунтов в warmup пуле → эффект сети (больше пул = лучше репутация каждого)
- **Community moat:** 2,000+ активных в Telegram → switching cost
- **Интеграции:** amoCRM/Bitrix24 → IT-инфраструктура клиентов завязана на Поток

## Module 5: Second-Order Thinking

### Immediate Effects (если запустим Deliverability-first стратегию)

- SDR/Основатели получают рабочий warmup для Яндекс → регистрируются
- Coldy видит нового конкурента с AI → ускоряет свой roadmap
- vc.ru/Telegram получают контент про cold email в РФ → органический трафик

### Secondary Effects (через 6–12 мес)

- **Позитивные:**
  - Warmup пул растёт экспоненциально: каждый новый пользователь усиливает всех остальных
  - Word-of-mouth: "inbox rate 89%+" → вирусный эффект в профессиональных сообществах
  - Агентства приходят сами, когда видят multi-client кабинет → ARPU × 3–5

- **Негативные (риски):**
  - Яндекс/Mail.ru может обнаружить warmup трафик и начать блокировки → нужны лимиты и вариативность
  - Coldy снижает цену до ₽1,500 → ценовая война
  - Плохие актёры используют платформу для спама → 38-ФЗ проблемы → нужен ToS с верификацией

### Time Horizons

| Горизонт | Состояние |
|----------|-----------|
| **6 месяцев** | 200 клиентов, warmup пул 800+ аккаунтов, MRR ~₽400K |
| **12 месяцев** | 600 клиентов, warmup пул 3000+ аккаунтов, MRR ~₽1.5M |
| **24 месяца** | 1500 клиентов, warmup пул 8000+ аккаунтов, MRR ~₽4M, launch AI Leads Database |
| **36 месяца** | Потенциальный выход или стратегическое партнёрство |

### Unintended Consequences

| Риск | Вероятность | Митигация |
|------|----------:|-----------|
| Яндекс блокирует warmup паттерны | 30% | Случайные delays, human-like patterns, rate limits |
| Крупный клиент использует платформу для спама | 40% | ToS enforcement, send rate caps, abuse detection |
| Instantly возвращается в РФ | 20% | Продать до этого или использовать как validation |
| Coldy привлекает инвестиции и копирует | 35% | Скорость + community + интеграции как барьеры |
| Технический долг при масштабировании | 60% | SPARC-документированная архитектура, тесты |

### Second-Order Validation: Warmup Network Effect

```
Пользователи ↑ → Аккаунтов в пуле ↑ → 
Качество warmup ↑ → Inbox Rate ↑ → 
Сарафанное радио ↑ → Пользователи ↑ (↻ positive loop)
```

Это именно тот flywheel, который делает Instantly непобедимым глобально. Мы строим тот же flywheel для РФ-рынка.

## Module 6: Contradiction Analysis (TRIZ)

### 6.1 Ideal Final Result (IFR)

```
Система ИДЕАЛЬНО:
- Прогревает email аккаунты для Яндекс/Mail.ru САМА СОБОЙ
- Без ручного участия пользователя
- Используя ТОЛЬКО существующие ресурсы (SMTP/IMAP протоколы, аккаунты пользователей)
- Гарантирует inbox rate ≥85% с первой попытки
- Стоит ₽0 маргинальных затрат на каждый дополнительный аккаунт
```

Мы не можем достичь IFR полностью, но он указывает направление: автоматизация через peer network.

### 6.2 Ключевые Противоречия

**Contradiction 1: Цена (Physical)**
> Цена должна быть **ВЫСОКОЙ** → для юнит-экономики (CAC recovery, операционные затраты)
> Цена должна быть **НИЗКОЙ** → для массовой adoption в РФ (чувствительность к цене)

**Contradiction 2: Warmup Pool (Technical)**
> Warmup пул должен быть **БОЛЬШИМ** → для высокого качества прогрева (статистика, разнообразие)
> Warmup пул **ПУСТОЙ** при запуске → chicken-and-egg, нет пользователей до запуска

**Contradiction 3: Простота vs Мощность (Technical)**
> Интерфейс должен быть **ПРОСТЫМ** → SDR/Основатели (не технари) должны настроить за 10 мин
> Функционал должен быть **МОЩНЫМ** → агентства нужен multi-client, тонкий контроль, reporting

**Contradiction 4: AI Reply Agent (Technical)**
> AI должен отвечать **АВТОНОМНО** → снимает 2–4 ч/день рутины
> AI должен отвечать **С СОГЛАСОВАНИЯ** → критично для B2B сделок, риск потери клиента

### 6.3 TRIZ Разрешение

**Contradiction 1 → TRIZ #35 (Parameter Change) + Separation in TIME:**

```
Бесплатно/дёшево → пока пользователь не получил Aha Moment (первое письмо в inbox)
Дорого/flat → после Aha, на эмоции победы
```

Реализация:
- Trial 7 дней бесплатно (или freemium с 1 аккаунтом)
- Paywall показывается сразу после первого inbox-попадания
- ₽1,990/мес Старт — ниже Coldy, выше порога "игрушка"
- ARPU растёт через апселл: Старт → Про → Агентство (не через повышение базовой цены)

**Contradiction 2 → TRIZ #10 (Preliminary Action) + #25 (Self-service):**

Chicken-and-egg решение через "Bootstrap Pool":
```
День 0: Создаём 50–100 seed аккаунтов (реальные аккаунты на Яндекс)
День 1: Первые пользователи добавляют свои аккаунты → пул = 100 seed + N user
День 30: 100 пользователей × 2 аккаунта = 200 user + 100 seed = 300 аккаунтов
```

Seed аккаунты — наши операционные затраты, но решают cold-start. Через 3 месяца они не нужны.

**Contradiction 3 → TRIZ #1 (Segmentation) + #17 (Another Dimension):**

```
Слой 1 (ПРОСТОЙ): Quick Setup Wizard → "Подключи аккаунт → Запусти warmup" — 3 шага
Слой 2 (МОЩНЫЙ): Advanced Settings → доступны, но скрыты за "Показать настройки"
Слой 3 (АГЕНТСКИЙ): Multi-client кабинет → отдельный раздел для agency tier
```

Разные интерфейсы для разных ролей. Каждый видит только то, что нужно его задаче.

**Contradiction 4 → TRIZ #23 (Feedback) + Separation by CONDITION:**

```
Autopilot: AI отвечает сам → если confidence > 85% И категория = "interested/meeting"
Draft Mode: AI пишет черновик → если confidence < 85% ИЛИ категория = "pricing/objection"
Manual: Уведомление пользователю → если категория = "angry/unsubscribe/legal"
```

Разные режимы для разных типов входящих. Пользователь контролирует threshold.

### 6.4 Резюме TRIZ-решений

| Противоречие | TRIZ Принцип | Решение |
|-------------|-------------|---------|
| Цена высокая/низкая | #35 + Time Separation | Freemium → Aha → Paywall |
| Пул большой/пустой | #10 + #25 | 100 seed-аккаунтов при запуске |
| Интерфейс простой/мощный | #1 + #17 | 3 слоя: Wizard / Advanced / Agency |
| AI автономный/контролируемый | #23 + Condition | Autopilot / Draft / Manual threshold |

## Module 7: Design Thinking — Human-Centered

### Empathize: Портрет пользователя (Segment A — 50% рынка)

**Алексей, 31 год, Co-Founder B2B SaaS, Москва**
- Команда 8 человек, нет выделенного SDR
- Задача: 30 qualified лидов/мес чтобы выполнить план по выручке
- Текущий инструмент: Яндекс.Почта вручную + таблицы Excel
- Боль: тратит 3 часа в день на написание и мониторинг холодных писем
- Триггер поиска: "CTO сказал — нам нужен pipeline, нанимать некого"
- Страх: "куплю подписку, а письма всё равно пойдут в спам"

**Наталья, 27 лет, менеджер аутрич-агентства, Санкт-Петербург**
- Ведёт 7 клиентов одновременно в разных вкладках браузера
- Боль: переключение между аккаунтами, ошибки при ручных ответах от имени клиента
- Триггер: "Клиент хочет отчёт, а у меня нет единого дашборда"

### Define (Reframe): Проблема глазами пользователя

```
"Мне нужен инструмент, который сделает так, чтобы мои письма 
приходили во входящие Яндекс АВТОМАТИЧЕСКИ, а я мог потратить 
своё время на разговоры с лидами, а не на настройку технологий"
```

Ключевое: **не "warmup"** (термин инструмента) — а **"мои письма попадают в inbox"** (результат).

### Ideate: Решения

1. **Радикально простое:** Кнопка "Запустить прогрев" → одно действие → готово
2. **Радикально умное:** AI анализирует ответы → предлагает следующий шаг для каждого лида
3. **Сетевой эффект:** "Пригласи 3 друга → получи +1 месяц" — вирусный рост через warmup invite

### Prototype (MVP для Aha Moment)

Минимальный путь к "письмо попало в inbox Яндекс":
```
1. Регистрация (email + пароль) — 1 мин
2. Подключение Яндекс аккаунта (SMTP/IMAP credentials) — 3 мин
3. Автоматическая DNS-проверка (SPF/DKIM/DMARC) — моментально
4. Кнопка "Запустить прогрев" — 1 клик
5. Dashboard: "Первое письмо отправлено/получено" — через 2–5 мин
6. Inbox Score gauge: начинает расти с первого часа
```

**Time to Aha: ≤10 минут от регистрации**

## Module 8: OODA Loop — Решение и действие

### Observe (текущие факты без интерпретации)

- 0 платящих клиентов
- 0 аккаунтов в warmup пуле
- Coldy имеет ~500–1000 клиентов, нет AI, нет multi-client
- SPARC документация создаётся прямо сейчас
- YooKassa SDK готов, SMTP/IMAP протоколы стандартны

### Orient (снимаем когнитивные искажения)

❌ "Нужно идеальный продукт перед запуском" → Нет. Нужен Aha Moment за 10 мин.
❌ "Нужно 1000 аккаунтов в warmup пуле сразу" → Нет. 100 seed аккаунтов достаточно для первых 50 пользователей.
❌ "AI Reply Agent — сложная фича" → Нет. Prompt Engineering + Claude API + state machine = 2 недели разработки.
✅ Community trust строится ДО продукта — Telegram-канал запускаем сейчас.

### Decide (высшего expected value решение)

**Решение:** MVP через 8 недель с одним WOW-моментом:

> "Подключи Яндекс аккаунт → через 10 минут первое тестовое письмо во входящих → Inbox Score растёт на глазах"

Это одно предложение = вся маркетинговая стратегия + продуктовая стратегия.

**Reversibility:** Двусторонняя дверь на всех решениях кроме seed аккаунтов (покупка). Пробуем быстро.

### Act (первые конкретные действия)

```
Неделя 1: Создать seed пул (50 Яндекс аккаунтов, App Passwords)
Неделя 1: Telegram-канал "Поток — холодный аутрич в РФ" → начать контент
Неделя 2–4: API + warmup service (Fastify + imapflow + BullMQ)
Неделя 4–6: Frontend (Next.js 15, onboarding, Inbox Score dashboard)
Неделя 6–8: YooKassa интеграция + Alpha тест с 10 beta пользователями
Неделя 8: Публичный запуск на vc.ru
```

## Module 9: Solution Synthesis — Стратегический план

### Интегрированные инсайты

| Модуль | Ключевой вывод |
|--------|---------------|
| First Principles | Warmup = SMTP/IMAP peer exchange. Нет технических барьеров, только организационные |
| Root Cause | Gap: глобальные ушли + локальные не достраивают полный стек |
| SCQA | Ключевой вопрос: как стать единственным full-stack решением в РФ |
| Game Theory | Войти с ценой ниже Coldy → взять весь новый спрос → data moat через warmup сеть |
| Second-Order | Positive flywheel: пользователи ↑ → пул ↑ → качество ↑ → рост ↑ |
| TRIZ | 4 противоречия разрешены: Freemium→Paywall, Seed pool, Слои UX, AI threshold |
| Design Thinking | Aha = "письмо во входящих Яндекс за 10 мин" → всё строится вокруг этого |
| OODA | Запуск через 8 недель, community-first, Telegram до продукта |

### Стратегия: 3 фазы

#### Фаза 1: Establish (M1–M3) — Deliverability Moat

**Цель:** 100 платящих клиентов, доказать Aha Moment работает

| Действие | Детали |
|---------|--------|
| Seed warmup пул | 100 Яндекс аккаунтов, App Passwords, BullMQ warmup jobs |
| MVP Warmup Core | SMTP/IMAP + Inbox Score + Dashboard |
| MVP Campaigns | Отправка с rate limiting, A/B subject lines |
| MVP Payments | YooKassa подписки (Старт ₽1,990) |
| Community | Telegram-канал, vc.ru статьи, YouTube туториалы |
| Запуск | Beta → Product Hunt RU → vc.ru публикация |

#### Фаза 2: Scale (M4–M9) — Agency + AI Moat

**Цель:** 500 платящих клиентов, warmup пул 3,000+ аккаунтов

| Действие | Детали |
|---------|--------|
| AI Reply Agent | Claude API + Autopilot/Draft/Manual режимы |
| Multi-client кабинет | Agency tier ₽9,990, N клиентов, unified inbox |
| amoCRM / Bitrix24 | REST API интеграции, lead sync |
| Contact Database | Парсинг открытых источников РФ (hh.ru, LinkedIn RU) |
| Affiliate program | 20% recurring, партнёры-агентства |

#### Фаза 3: Dominate (M10–M18) — Data Moat

**Цель:** $1M ARR, стать стандартом для cold email в РФ

| Действие | Детали |
|---------|--------|
| AI Leads Database | РФ-специфичная B2B база, SuperSearch аналог |
| Telegram Outreach | Multichannel: email + Telegram одновременно |
| API для разработчиков | Webhook + REST API → n8n/Zapier/Albato интеграции |
| Enterprise | On-premise версия для крупных корпораций (152-ФЗ data residency) |

### Метрики успеха

|---------|:--:|:--:|:---:|

| NPS | >30 | >45 | >60 |

### Приоритеты риск-митигации

| Риск | Действие |
|------|---------|
| Яндекс детектирует warmup | Human-like delays (random 30s–5min), разные subject lines, дневные лимиты |
| Спам-актёры | ToS верификация, send rate cap 200/день для новых аккаунтов, abuse detection |
| Ценовая война Coldy | Держать цену, конкурировать по features (AI, multi-client) а не по цене |
| Технический долг | SPARC docs + тесты + code review по каждой фиче |
| 38-ФЗ штраф | Обязательная кнопка отписки, ToS запрет массового спама, персонализация |
